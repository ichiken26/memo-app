# 画面仕様書

## /

Google 認証前のログイン画面と、認証後のトップ画面を表示する。

- 認証前は「Google でログイン」ボタンを表示する。
- 認証後はログインユーザーに紐づくメモだけを、タグごとに整理されたプレビュー一覧で表示する。
- タグ名のある見出し要素をクリックすると `/tag/{タグID}?tag={タグ名}` に遷移する。
- メモプレビューをクリックすると `/memo/{メモID}` に遷移する。
- 作成ボタンから `/memo/new` に遷移する。
- 検索ボタンから `/search` に遷移する。

## /search

検索画面を表示する。

- 検索窓をクリックすると、Google の検索窓のように過去の検索履歴候補を表示する。
- 検索条件は検索ワードとタグの複数選択で構成する。
- 検索対象はログインユーザーに紐づくメモのみとする。
- 検索ワードに空白区切りで複数語を入力した場合、すべての語を含むメモを部分一致で検索する。
- 検索実行後、URL に `?tag={タグ名}&search_word={検索ワード}` を保持する。
- 複数タグを指定した場合、`tag` クエリはカンマ区切りで保持する。
- API リクエストボディではタグの `id` と `name` のセット、および `search_word` を送る。
- API に送信するタグ ID は `tag-{生成日}-{タグ名}` 形式とする。
- 検索結果が 0 件の場合、検索結果表示エリアに「該当するメモは存在しません」と表示する。

例:

```text
/search?tag=Research,Meeting&search_word=API
```

## /memo/new

メモ作成画面を表示する。

- タイトル、本文、タグ選択欄を表示する。
- タグ選択欄では既存タグを選択できる。
- `+` ボタンから新しいタグを追加できる。
- 新規追加したタグは作成中のメモに割り当てられる。
- 保存ボタンでメモを作成し、作成後は `/memo/{メモID}?tag={タグ名}` に遷移する。
- 作成したメモはログインユーザーのFirebase `uid` に紐づける。

## /memo/{メモID}

メモ詳細兼編集画面を表示する。

- URL 形式は `/memo/{メモID}?tag={タグ名}`。
- タイトル、本文、更新日、保持しているすべてのタグを表示する。
- タイトルと本文は Google Keep のようにその場で編集できる。
- タグ選択欄では既存タグの追加・解除と新規タグ追加ができる。
- 入力変更後 1 秒間追加変更がなければ自動保存する。
- 保存ボタンでも手動保存できる。
- 保存未完了の変更がある状態でページ離脱しようとした場合は警告する。
- 存在しないメモ ID の場合は「メモが見つかりません」と表示する。
- ログインユーザーに紐づかないメモ ID は存在しないメモとして扱う。

例:

```text
/memo/memo-20260512-タグ検索-UI-の検討?tag=Product,Research
```

## /tag/{タグID}

タグ別メモ一覧画面を表示する。

- URL 形式は `/tag/{タグID}?tag={タグ名}`。
- 指定タグを保持しているメモ一覧を表示する。
- 表示対象はログインユーザーに紐づくメモのみとする。
- メモプレビューをクリックすると `/memo/{メモID}` に遷移する。
- メモプレビュー内のタグチップをクリックすると該当タグの一覧へ遷移する。
- 存在しないタグ ID の場合は「指定されたタグ ID に一致するタグはありません。」と表示する。

例:

```text
/tag/tag-20260501-Research?tag=Research
```
